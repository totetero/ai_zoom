# レスポンシブデザイン詳細仕様

ブラウザの画面サイズ変更（リサイズ）や、モバイルデバイス・デスクトップデバイスの違いに柔軟に対応するための詳細な設計仕様です。

## 1. デザイン原則
- **Fluid & Adaptive:** 単に「縮小」するのではなく、各画面幅において最適な情報量と操作性を提供します。
- **Interactive Consistency:** どのデバイスでも「無限ズーム」という核となる体験が一貫していることを重視します。

## 2. ブレイクポイントとレイアウト戦略

| デバイス | 画面幅 (px) | レイアウトの特徴 |
| :--- | :--- | :--- |
| **Mobile** | < 768 | 垂直スタック、コントロールの簡略化、ターゲットの拡大 |
| **Tablet/Small PC** | 768 - 1024 | デスクトップの要素を維持しつつ、余白を調整 |
| **Desktop/Large** | >= 1024 | 全ての情報を四隅に配置、リッチなタイポグラフィ |

### 2.1. UI要素の調整
- **年表示 (Year Title):** 
  - `clamp(3rem, 10vw, 6rem)` 等を使用し、画面の主役であることを維持しつつ画面幅に追従させます。
- **ボタン類 (Editor/Processor):**
  - モバイルでは上部にコンパクトに配置、または「設定」メニューアイコン内に集約します。
- **被写体選択 (Subject Selector):**
  - モバイルでは画面上部、デスクトップでは左上に配置。要素が多い場合は横スクロール可能にします。
- **コントロール (Slider):**
  - モバイルでは画面下部の広い範囲（幅 90% 以上）を占有し、操作ミスを防ぎます。

## 3. 技術的実装
### 3.1. CSS (Vanilla CSS)
- **CSS変数:** カラーパレットやフォントサイズ、余白の基本単位を定義し、メディアクエリで一括して調整できるようにします。
- **Flex/Grid:** 絶対配置 (`position: absolute`) に頼りすぎず、コンテナ内での適切な配置を行います。

### 3.2. Canvas (Three.js)
- **リサイズ検知:** `@react-three/fiber` のデフォルトの挙動（親要素に追従）を活かしつつ、計算ロジック内で `size` (width/height) が変更された際に、アスペクト比に基づいた描画範囲の再計算を即座に行います。
- **Camera Aspect:** `OrthographicCamera` の `left`, `right`, `top`, `bottom` をリサイズ後の `width`, `height` に合わせて更新します。

### 3.3. 操作系 (Hooks/Event)
- **Wheel & Touch:** マウスホイールだけでなく、画面全体のタッチ・ドラッグジェスチャを進捗（Progress）に反映させるためのイベントハンドラを追加検討します。

## 4. 検証項目
- [ ] ブラウザのウィンドウをドラッグして縮めた際に、UI要素が重ならないか。
- [ ] iOS/Android のモバイルブラウザで、ボタンがストレスなく押せるか。
- [ ] 画面回転（ポートレート/ランドスケープ）時に Canvas が適切に再描画されるか。
- [ ] 小さな画面でもメッセージが読み取れるか。
