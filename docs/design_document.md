# 娘の誕生日写真 インフィニットズームWebアプリ 設計図

## 1. システムアーキテクチャ
フロントエンドのみで完結する静的Webアプリケーション（SPA）。将来の動画エクスポートを前提に描画レイヤーとして Canvas を利用する。

- **言語:** TypeScript
- **フレームワーク:** React
- **描画・アニメーション制御:** WebGL (React Three Fiber / Three.js)
- **ビルドツール:** Vite

## 2. ディレクトリ構成
```text
src/
├── assets/
│   ├── img/            # 写真ファイル群 (0.jpg, 1.jpg, ...)
│   └── data/
│       └── frames.json # 各写真のメタデータ（額縁の座標、角度など）
├── components/
│   ├── ZoomCanvas.tsx  # インフィニットズームの描画とカメラ操作を行うメインCanvas
│   ├── Overlay.tsx     # 年齢やメッセージを浮き出させて表示するDOM UI
│   └── Controls.tsx    # 再生・停止、シークバーなど
├── tools/
│   └── FrameEditor.tsx # 課題解決用: 画像の額縁座標を取得するための開発用ヘルパーツール画面
├── hooks/
│   ├── usePreloadImages.ts # 複数枚の画像をCanvasに転送する前のプリロード管理
│   └── useZoomContext.ts   # アニメーション進行度やカメラの状態管理用フック
├── App.tsx             # アプリのルートコンポーネント
└── main.tsx            # エントリポイント
```

## 3. データ構造設計 (frames.json)
各写真ごとの額縁の位置（ズームの目標地点）を定義します。

```json
[
  {
    "id": 0,
    "filename": "0_birth.jpg",
    "year": 2020,
    "age": 0,
    "message": "誕生",
    "points": null 
  },
  {
    "id": 1,
    "filename": "1_year.jpg",
    "year": 2021,
    "age": 1,
    "message": "1歳の誕生日",
    "points": [
      { "x": 300.0, "y": 450.0 },
      { "x": 450.0, "y": 420.0 },
      { "x": 420.0, "y": 620.0 },
      { "x": 270.0, "y": 650.0 }
    ] // Outer画像の解像度上での、対象となる額縁の4隅の座標（左上から時計回りなど）
  }
]
```

## 4. コンポーネント設計＆機能詳細

### 4.1. ZoomCanvas.tsx (コアコンポーネント)
- **役割:** 複数の画像を単一の WebGL Canvas (`@react-three/fiber`) 要素上に配置し、カメラを操作して描画を行う。
- **実装方針:**
  - 単一の `<Canvas>` 要素を使用し、`useFrame` ループ内で Three.js のカメラ (`OrthographicCamera`) のパラメータを操作する。
  - `frames.json` の `points` 座標を取得し、「射影変換（ホモグラフィ）行列」を計算。内側の画像を外側の画像の額縁の4点にパースペクティブを合わせてピッタリと矩形変形（3D平面描画）してレンダリングする。
  - トランジション時にはカメラの逆変換行列を計算し、数学的に均等なスピードで対数・指数イージングをかけながら額縁内へ滑らかに飛び込むようにズームする。
  - **透明度制御 (2026-04-16 更新)**: 写真が切り替わる際の違和感をなくすため、進捗 `p` に連動した透明度（Opacity）調整を行う。中心画像はズーム終了直前にフェードアウトし、背景画像はズーム開始直後にフェードインすることで、切り替わりの瞬間を滑らかにする。

### 4.2. FrameEditor.tsx (開発時専用ヘルパー)
- **役割:** 写真の額縁の4つ角（左上、右上、右下、左下）をドラッグ＆ドロップで指定し、その座標をJSON配列として出力する画面。
- **リデザイン方針 (2026-03-27):** 
  - 従来の「順番にクリック」する方式を廃止し、デフォルト配置されたハンドルをマウス操作で動かす方式に変更。
  - **2026-03-29 更新**: 画像の境界線外にもマーカーを配置できるよう、キャンバスにパディング（余白）を追加し、ドラッグ操作の改善と座標変換の補正を実施。
  - 詳細は [FrameEditor_Redesign.md](file:///Users/tote/project/git/totetero/ai_zoom/docs/FrameEditor_Redesign.md) を参照。

### 4.3. useZoomContext.ts
- **役割:** アニメーションの進行度（progress: 0.0 〜 1.0 または 年数インデックス）を管理し、現在どの写真からどの写真に向かって遷移しているかを計算する。
- **実装方針:**
  - 現在の進行度合をもとに、画面全体のカメラビューを構成する行列（Matrix4）を計算するための補間処理などを担当・補助する。

## 5. 演出アルゴリズムの考え方
1. 画像A（現在）の中に、画像B（過去）が入れ子になって描画されている。
2. カメラ位置ベクトルが画像Aをズームインしていくと、画面の中央が徐々に額縁の部分に寄り、世界全体が拡大される。
3. 同時に、額縁の傾き（rotation）を相殺するようにカメラのCanvas Context自体が回転し、最終的に画像Bが画面に対して水平・垂直かつフルスクリーンになるようにする。
4. この状態になったら、完全に画像Bの世界へとフォーカスが移る。これの連続によりインフィニットズームを表現する。
5. **スムーズな遷移のためのフェード処理 (2026-04-16 追加)**:
   - ズームの進捗（境界）での「パッ」とした切り替えを抑えるため、以下の計算式に基づき透明度を調整する。
   - 進捗 `p` (0.0~1.0) に対し、しきい値 `threshold` (デフォルト 0.2) を用いる。
   - **背景画像 (Outer)**: `min(1.0, p / threshold)` により、開始直後にフェードイン。
   - **中央画像 (Inner)**: `min(1.0, (1.0 - p) / threshold)` により、終了直前にフェードアウト。
   - これにより、世界が入れ替わる瞬間に古い画像が消え、新しい背景が自然に現れる演出を実現している。

- **Remotion連携プラン:** アニメーションのパラメータ管理と描画ロジックを独立させておき、将来的に Remotion 等でのフレームバイフレームのレンダリングを行う仕組みに載せ替えられるようなステート設計を採用する。
