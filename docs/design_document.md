# 娘の誕生日写真 インフィニットズームWebアプリ 設計図

## 1. システムアーキテクチャ
フロントエンドのみで完結する静的Webアプリケーション（SPA）。

- **言語:** TypeScript
- **フレームワーク:** React
- **アニメーション制御:** GSAP (または CSS Transform + requestAnimationFrame)
- **ビルドツール:** Vite

## 2. ディレクトリ構成
```text
src/
├── assets/
│   ├── img/            # 写真ファイル群 (0.jpg, 1.jpg, ...)
│   └── data/
│       └── frames.json # 各写真のメタデータ（額縁の座標、角度など）
├── components/
│   ├── ZoomViewer.tsx  # インフィニットズームを処理するメインコンポーネント
│   ├── Overlay.tsx     # 年齢やメッセージの表示
│   └── Controls.tsx    # 再生・停止、シークバーなど
├── tools/
│   └── FrameEditor.tsx # 課題解決用: 画像の額縁座標を取得するための開発用ヘルパーツール画面
├── hooks/
│   ├── usePreloadImages.ts # 画像プリロード用カスタムフック
│   └── useZoomAnimation.ts # 動的アニメーション制御用フック
├── App.tsx             # アプリのルートコンポーネント
└── main.tsx            # エントリポイント
```

## 3. データ構造設計 (frames.json)
各写真ごとの額縁の位置（ズームの目標地点）を定義します。

```json
[
  {
    "id": 0,
    "filename": "0_birth.jpg",
    "year": 2020,
    "age": 0,
    "message": "誕生",
    "frameRect": null 
  },
  {
    "id": 1,
    "filename": "1_year.jpg",
    "year": 2021,
    "age": 1,
    "message": "1歳の誕生日",
    "frameRect": {
      "x": 300,        // 1歳の写真(1024x768とする)に対する、額縁のX座標
      "y": 450,        // Y座標
      "width": 150,    // 額縁の幅
      "height": 200,   // 額縁の高さ
      "rotation": -5   // 額縁の傾き（度）
    }
  }
]
```
*注: `frameRect` は、現在の写真（例: 1歳の写真）内において、前回の写真（例: 0歳の写真）が配置されている額縁の相対座標と回転角度を示します。*

## 4. コンポーネント設計＆機能詳細

### 4.1. ZoomViewer.tsx (コアコンポーネント)
- **役割:** 複数の画像を階層的に重ねて配置し、カメラ（あるいはコンテナ全体）のスケールと回転を操作する。
- **実装方針:**
  - 全ての画像を絶対配置(`position: absolute`)で重ねる。
  - `frames.json` の `frameRect` を計算し、画像を初期ロード時に「額縁の中にピッタリ収まるようにサイズと位置・角度を合わせて配置」する。
  - ユーザー操作（スクロールやボタン）に応じて、親要素全体を拡大・回転させ、カメラが額縁の中に飛び込むようなトランジションを行う。
  - DOMの肥大化を防ぐため、現在表示領域から完全に外れた巨大/微小過ぎる画像は一時的に非表示（`display: none`）にする。

### 4.2. FrameEditor.tsx (開発時専用ヘルパー)
- **役割:** 写真の額縁の4つ角（左上、右上、右下、左下）をマウスクリックで指定し、その座標から `x, y, width, height, rotation` を計算して `frames.json` 用のJSON文字列を出力する画面。

### 4.3. useZoomAnimation.ts
- **役割:** アニメーションの進行度（progress: 0.0 〜 1.0 または 年数インデックス）を管理し、現在どの写真からどの写真に向かって遷移しているかを計算する。
- **実装方針:**
  - 現在の目標となるindex（年数）と、現在の進行度合をもとに、スケール値と回転値を線形（またはイーズアウトなどの関数で）補間して CSS の `transform` として出力する。

## 5. 演出アルゴリズムの考え方
1. 画像A（現在）の中に、画像B（過去）を持つ額縁がある。
2. カメラが画像Aをズームインしていくと、額縁の部分が画面全体に拡大される。
3. 同時に、額縁の傾き（rotation）に合わせてカメラも回転し、最終的に画像Bが画面に対して水平・垂直かつフルスクリーンになるようにする。
4. この状態になったら、現在のフォーカスが完全に画像Bに切り替わる。これを連続させることでインフィニットズームを表現する。
