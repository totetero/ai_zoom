# 娘の誕生日写真 インフィニットズームWebアプリ 設計図

## 1. システムアーキテクチャ
フロントエンドのみで完結する静的Webアプリケーション（SPA）。将来の動画エクスポートを前提に描画レイヤーとして Canvas を利用する。

- **言語:** TypeScript
- **フレームワーク:** React
- **描画・アニメーション制御:** HTML5 Canvas API + requestAnimationFrame (または React対応のCanvasライブラリ)
- **ビルドツール:** Vite

## 2. ディレクトリ構成
```text
src/
├── assets/
│   ├── img/            # 写真ファイル群 (0.jpg, 1.jpg, ...)
│   └── data/
│       └── frames.json # 各写真のメタデータ（額縁の座標、角度など）
├── components/
│   ├── ZoomCanvas.tsx  # インフィニットズームの描画とカメラ操作を行うメインCanvas
│   ├── Overlay.tsx     # 年齢やメッセージを浮き出させて表示するDOM UI
│   └── Controls.tsx    # 再生・停止、シークバーなど
├── tools/
│   └── FrameEditor.tsx # 課題解決用: 画像の額縁座標を取得するための開発用ヘルパーツール画面
├── hooks/
│   ├── usePreloadImages.ts # 複数枚の画像をCanvasに転送する前のプリロード管理
│   └── useZoomContext.ts   # アニメーション進行度やカメラの状態管理用フック
├── App.tsx             # アプリのルートコンポーネント
└── main.tsx            # エントリポイント
```

## 3. データ構造設計 (frames.json)
各写真ごとの額縁の位置（ズームの目標地点）を定義します。

```json
[
  {
    "id": 0,
    "filename": "0_birth.jpg",
    "year": 2020,
    "age": 0,
    "message": "誕生",
    "frameRect": null 
  },
  {
    "id": 1,
    "filename": "1_year.jpg",
    "year": 2021,
    "age": 1,
    "message": "1歳の誕生日",
    "frameRect": {
      "x": 300,        // 1歳の写真の原寸(例えば1024x768とする)に対する額縁のX座標
      "y": 450,        // Y座標
      "width": 150,    // 額縁の幅
      "height": 200,   // 額縁の高さ
      "rotation": -5   // 額縁の傾き（度）
    }
  }
]
```

## 4. コンポーネント設計＆機能詳細

### 4.1. ZoomCanvas.tsx (コアコンポーネント)
- **役割:** 複数の画像を単一の Canvas 要素上に配置し、スケールと回転を操作して描画を行う。将来的な MediaRecorder による動画化の対象要素となる。
- **実装方針:**
  - 単一の `<canvas>` 要素を使用し、すべての写真描画を `requestAnimationFrame` ループ内で `ctx.drawImage` 等を用いて行う。
  - `frames.json` の `frameRect` を計算し、各々の写真が「１つ外側の（新しい年の）写真の額縁の中にピッタリ収まるように」内部的な座標、サイズ、回転を計算してレンダリングする。
  - ユーザー操作（または自動再生）に応じて、Canvas のコンテキスト(`ctx.translate`, `ctx.scale`, `ctx.rotate`)を操作し、カメラが額縁の中に飛び込むトランジションを行う。
  - パフォーマンスを保つため、カメラが拡大されすぎたり、逆に小さくなりすぎて見えなくなった画像はループ内で `drawImage` をスキップする（カリング）。

### 4.2. FrameEditor.tsx (開発時専用ヘルパー)
- **役割:** 写真の額縁の4つ角（左上、右上、右下、左下）をマウスクリックで指定し、その座標から `x, y, width, height, rotation` を計算して `frames.json` 用のJSON文字列を出力する画面。

### 4.3. useZoomContext.ts
- **役割:** アニメーションの進行度（progress: 0.0 〜 1.0 または 年数インデックス）を管理し、現在どの写真からどの写真に向かって遷移しているかを計算する。
- **実装方針:**
  - 現在の目標となるindex（年数）と、現在のタイムラインの進行度合をもとに、スケール値と回転値を線形（またはイーズアウトなどの関数で）補間して、毎フレームごとの Canvas の Transformation Parameter （x, y, scale, rotation）を算出する。

## 5. 演出アルゴリズムの考え方
1. 画像A（現在）の中に、画像B（過去）が入れ子になって描画されている。
2. カメラ位置ベクトルが画像Aをズームインしていくと、画面の中央が徐々に額縁の部分に寄り、世界全体が拡大される。
3. 同時に、額縁の傾き（rotation）を相殺するようにカメラのCanvas Context自体が回転し、最終的に画像Bが画面に対して水平・垂直かつフルスクリーンになるようにする。
4. この状態になったら、完全に画像Bの世界へとフォーカスが移る。これの連続によりインフィニットズームを表現する。

## 6. 動画エクスポートへの拡張 (Future Work)
- **MediaRecorder API:** `<canvas>` から `captureStream()` を呼び出し、録画開始ボタン押下時にそのストリームを保存し `.webm` や `.mp4` 形式（ブラウザ依存）で提供する。
- **Remotion連携プラン:** アニメーションのパラメータ管理と描画ロジックを独立させておき、将来的に Remotion 等でのフレームバイフレームのレンダリングを行う仕組みに載せ替えられるようなステート設計を採用する。
