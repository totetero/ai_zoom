# FrameEditor のリデザイン設計書

## 1. 現状の課題
現在の `FrameEditor.tsx` は、画像を4箇所順番にクリックすることで座標を指定する方式を採用しています。しかし、この方式には以下の課題があります：
- 一度クリックした位置の微調整ができない。
- 入力の順番（左上→右上→右下→左下）を間違えると最初からやり直しになる。
- 直感的な操作感に欠ける。

## 2. 解決策：ドラッグ＆ドロップ方式への移行
点（頂点）を「打つ」のではなく、あらかじめ配置された「ハンドルを動かす」方式に変更します。

### 2.1. デフォルト配置の導入
画像を開いた際、`points` データが存在しない場合は、画像の中央 50% を覆う矩形をデフォルトとして自動生成します。
- Top-Left: (25%, 25%)
- Top-Right: (75%, 25%)
- Bottom-Right: (75%, 75%)
- Bottom-Left: (25%, 75%)

### 2.2. インタラクティブな操作
- **MouseDown**: 頂点の付近（ヒット判定範囲内）をクリックした際に、その頂点を「掴んだ」状態にします。
- **MouseMove**: 掴んでいる頂点をポインタに合わせてリアルタイムに移動させ、キャンバス描画を更新します。
- **MouseUp**: 移動を確定し、内部データ（allData）と出力用の JSON テキストを更新します。

### 2.3. 視覚的フィードバック
- 頂点の上にマウスがある時はカーソルを `pointer` または `crosshair` に、ドラッグ中は `grabbing` に変更します。
- 掴んでいる頂点は色を変化（例：赤から黄色）させ、選択状態を強調します。

### 2.5. 画像外への配置サポート (2026-03-29 更新)
ズームの構図によっては、内側の画像を外側の画像の境界線ギリギリ、あるいは一部はみ出すように配置したい場合があります。これを可能にするため、以下の機能を導入しました：
- **キャンバスパディング**: 画像の周囲に 20% の余白を追加し、画像の外側にもマーカーを配置できるようにしました。
- **画像境界の可視化**: 画像の本来の範囲を示すガイド線を描画し、直感的なレイアウトを支援します。
- **ウィンドウレベルのドラッグ**: マウスがキャンバス外に出てもドラッグが継続されるよう、イベント管理を改善しました。

## 3. 実装の主な変更点
- `handleCanvasClick` を廃止し、`handleMouseDown`, `handleMouseMove`, `handleMouseUp` を実装。
- `useEffect` 内で初期配置が存在しない場合のデフォルト生成ロジックを追加。
- 描画ループ（Canvas描画）において、ドラッグ中の状態を反映。
- `currentIndex`（現在の画像インデックス）の変更を監視し、切り替え後の画像に紐付く `points` データを自動的にステートへ同期。
- **座標変換ロジックの修正**: キャンバス上の座標を画像上のピクセル座標に変換する際、パディング分を差し引くことで、画像外の座標（負の値やサイズ超の値）を正しく保持。

### 2.4. 画像切り替えと状態同期
- **Next/Prev ボタン**: `currentIndex` を変更します。
- **useEffect による同期**: `currentIndex` の変更を検知し、`allData` から該当するフレームの座標を取得して `points` ステートを更新します。これにより、画像ごとに独立した点の位置が正しく表示されます。
- **データの永続化**: 各画像で調整された座標は、次の画像へ移動する前、または調整後に `allData` に保持され、最終的に JSON 形式で出力可能です。

## 4. 期待される効果
- 座標の細かな調整が可能になり、作業効率が向上する。
- 操作ミスによるやり直しが減り、ストレスのない編集体験を提供できる。
