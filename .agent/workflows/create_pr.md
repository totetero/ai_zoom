# Pull Request Creation Workflow

このワークフローは、現在の作業ブランチからデフォルトブランチ（通常は `main`）へのプルリクエスト（PR）を、AIによるタイトルと本文の自動生成を含めて作成するためのものです。

## 実行手順

1. **ブランチの確認**:
   現在のブランチ名を取得し、PRの作成元（Source）を確認します。
   ```bash
   git branch --show-current
   ```

2. **デフォルトブランチの特定**:
   通常は `main` または `master` を対象とします。リポジトリの設定を確認し、ベースとなるブランチ（Target）を決定します。

3. **変更内容（Diff）の取得**:
   ベースブランチとの差分を取得し、どのような変更が含まれているかを分析します。
   ```bash
   git diff main...HEAD
   ```

4. **ブランチ名の最終確認とリネーム**:
   取得した差分内容をもとに、現在のブランチ名が適切か確認します。
   もし現在のブランチ名に `_` （アンダースコア）が含まれていない（例: `feature/202603271015`）場合は、内容を反映したケバブケースのタスク概要を末尾に追加してリネームします。
   
   // turbo
   ```bash
   # ローカルブランチのリネーム
   git branch -m feature/YYYYMMDDHHmm_task-summary
   # リモートへのプッシュ（必要に応じて）
   git push -u origin feature/YYYYMMDDHHmm_task-summary
   ```

5. **PRタイトルと本文の生成**:
   取得した差分をもとに、以下の内容を日本語で生成します。
   - **タイトル**: 変更内容を端的に表す、分かりやすいタイトル。
   - **本文**: 以下の要素を含む詳細な説明（Markdown形式）。
     - **概要**: このPRが何を解決/実装するものか。
     - **変更点**: 具体的にどのような変更を行ったか。
     - **影響範囲**: 他のモジュールやUIへの影響、動作確認済みの内容など。

6. **PRの作成**:
   生成したタイトルと本文を使用して、GitHub CLI でPRを作成します。
   
   PR情報を一時ファイル（例: `/tmp/pr_body.md`）に保存し、以下のコマンドを実行します。
   // turbo
   ```bash
   gh pr create --title "<生成されたタイトル>" --body-file /tmp/pr_body.md
   ```
   ※ ドラフトとして作成したい場合は `--draft` オプションを追加してください。

7. **完了報告**:
   PRが作成されたことをユーザーに報告し、作成されたPRのURLを共有します。
