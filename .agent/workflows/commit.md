# Git Commit Automation Workflow

> [!IMPORTANT]
> このワークフローにおいて、エージェント（AI）は**自動的に `git add` を実行してはなりません**。
> ユーザーが自身でステージングしたファイル（`git add` 済みのファイル）のみをコミットの対象としてください。

このワークフローは、ステージング済みのファイルの変更内容をAIが分析し、適切なコミットメッセージを生成してコミットを完了させるための手順をガイドするものです。

## 実行手順

1. **ステージング済みのファイルの確認**:
   // turbo
   `git diff --cached --name-only` を実行し、コミット対象のファイルがあるか確認します。
   - もし何もなければ、ユーザーに「コミットするファイルがステージングされていません。コミットしたいファイルを `git add` してから再度実行してください」と伝え、処理を終了します。
   - **絶対にエージェントが自ら `git add .` などを行ってファイルをステージングしてはいけません。**

2. **変更内容の取得**:
   // turbo
   `git diff --cached` を実行し、全変更の詳細を取得します。

3. **コミットメッセージの生成**:
   取得した差分をもとに、以下のルールに従ってコミットメッセージを作成します。
   - 型（Type）をプレフィックスとして付ける（例: `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`, `chore:`）
   - 1行目: 50文字以内で、変更内容を簡潔に要約
   - 2行目: 空行
   - 3行目以降: 必要に応じて、変更の理由や詳細な説明を記述
   - **出力は日本語で作成する**

4. **コミットの実行**:
   生成したメッセージを使用してコミットを行います。コミットメッセージにクォートが含まれる場合は適切にエスケープするか、一時ファイルを経由してください。
   ```bash
   git commit -m "<生成されたコミットメッセージ>"
   ```

5. **完了報告**:
   コミットが正常に完了したことをユーザーに報告し、生成したコミットメッセージの内容を共有します。
