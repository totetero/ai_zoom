# Pull Request Review Workflow

このワークフローは、指定された GitHub Pull Request（PR）の差分を取得し、コードレビューを実施して、その結果を安全にコメントとして自動投稿するためのものです。

## 実行手順

1. ユーザーから対象の Pull Request 番号（または URL）を受け取る。
2. 作業中のリポジトリのディレクトリで、以下のコマンドを実行し PR の変更差分（Diff）を取得する。
   ```bash
   gh pr diff <PR番号>
   ```
3. 取得した Diff をもとに、以下の観点で精密なコードレビューを実施する。
   - 不要な初期コード（ボイラープレート）やログ、コメントアウトが残っていないか
   - ロジックや構造に問題はないか、ベストプラクティスに沿っているか
   - 良い変更点は積極的に「素晴らしい」と評価し、改善点があれば建設的に提案する
   - **出力は必ず、丁寧で正確な日本語のマークダウン形式で行うこと（※ユーザーのグローバルルール）**
   - **必ずレビューの末尾に、どの AI エージェント / モデル（例: Antigravity など）を使用してレビューを行ったかの署名（フッター）を追記する**
4. レビュー結果（全体評価、良かった点、改善点、結論など）を一時ファイル（例: `/tmp/pr_review_body.md`）に書き出す。（`write_to_file` ツールを使用）
5. ユーザー自身が作成した PR の場合、`--approve` フラグを使用するとエラーが発生するため、以下のコマンドを使用して**通常のコメントとしてレビュー内容を投稿**する。
   // turbo
   ```bash
   gh pr comment <PR番号> -F /tmp/pr_review_body.md
   ```
6. コマンドの実行が完了したら、投稿されたコメントの URL をユーザーに共有し、処理の完了を報告する。
