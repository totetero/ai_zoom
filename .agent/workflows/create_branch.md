# Create Branch from Main Workflow

このワークフローは、`main` ブランチの最新状態から新しい作業用のブランチを作成する手順をガイドするものです。

## 実行手順

1. **現在の状態の確認**:
   作業ディレクトリに未コミットの変更がないか（`git status`）確認します。変更がある場合は、変更をコミット・破棄・あるいはスタッシュ（`git stash`）で退避させるかユーザーに確認します。

2. **main ブランチの最新化**:
   `main` ブランチに切り替え、リモートの状態を反映します。
   // turbo
   ```bash
   git checkout main
   git pull origin main
   ```

3. **新しいブランチの作成**:
   ユーザーが指定した名前、またはタスクの内容を反映した適切なブランチ名で新しいブランチを作成します。
   **【ブランチ命名規則】**
   既存のブランチ構成に基づき、以下の命名規則を適用してブランチ名を決定してください。
   - 形式: `feature/YYYYMMDDNN_ケバブケースのタスク概要`
   - 例: `feature/2026032601_initial-development`、`feature/2026032602_readme-update`
     * `YYYYMMDD` は作業日の日付
     * `NN` はその日の連番（`01`, `02`, ...）
     * `_` （アンダースコア）で区切る
     * その後は英小文字とハイフンを使った簡潔な説明（ケバブケース）
   
   // turbo
   ```bash
   git checkout -b <規則に従った新しいブランチ名>
   ```

4. **完了報告**:
   ブランチの作成が正常に完了したことをユーザーに報告し、次の指示を待ちます。
