# Quick Fix Workflow

このワークフローは、`main` ブランチで行った簡単な修正を、新しいブランチに移動し、コミット、プルリクエスト作成、そして自動レビューまでを一気通貫で実行するためのものです。

## 実行手順

1. **ステージング状態の確認**:
   // turbo
   `git diff --cached --name-only` を実行し、コミット対象のファイルがステージングされているか確認します。
   - 何もステージングされていない場合は、ユーザーに「修正内容を `git add` してから再度実行してください」と伝え、終了します。
   - **AIが自動で `git add` を行うことは禁止されています。**

2. **作業ブランチの作成**:
   取得した変更内容をもとに、適切なブランチ名を作成して切り替えます。
   - **命名規則**: `feature/YYYYMMDDXX_task-summary`
     - `YYYYMMDD`: 本日の日付
     - `XX`: 連番（01, 02...）
     - `task-summary`: 修正内容を表すケバブケースの短い単語
   - **実行コマンド**:
     ```bash
     git checkout -b <生成したブランチ名>
     ```

3. **コミットの実行**:
   変更内容（`git diff --cached`）を分析し、適切なコミットメッセージを作成してコミットします。
   - メッセージ形式: `型: 概要`（日本語）
   - **実行コマンド**:
     ```bash
     git commit -m "<生成したコミットメッセージ>"
     ```

4. **プルリクエストの作成**:
   作成したブランチをリモートにプッシュし、プルリクエストを作成します。
   - **実行コマンド**:
     // turbo
     ```bash
     git push -u origin HEAD
     ```
   差分に基づき、日本語でタイトルと本文を生成し、GitHub CLI で PR を作成します。
   - **実行コマンド**:
     ```bash
     gh pr create --title "<生成したタイトル>" --body "<生成した本文>"
     ```

5. **自動レビューの実行**:
   作成された PR の番号を取得し、差分を分析してレビューコメントを投稿します。
   - **PR番号の取得**:
     // turbo
     ```bash
     gh pr view --json number -q .number
     ```
   - **差分の確認**:
     // turbo
     ```bash
     gh pr diff <取得した番号>
     ```
   - **レビューの投稿**:
     レビュー内容を `/tmp/pr_review_body.md` に保存した後、以下を実行します。
     ```bash
     gh pr comment <取得した番号> -F /tmp/pr_review_body.md
     ```

6. **完了報告**:
   PR の URL と、投稿したレビューの概要をユーザーに報告して終了します。
